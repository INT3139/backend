# CLAUDE.md

## Project Overview
This is the backend for the HRM (Human Resource Management) system of UET (University of Engineering and Technology, VNU). The system manages lecturers' profiles, recruitment, salary, rewards, workloads, and more.

### Tech Stack
- **Framework:** Express (TypeScript)
- **Database:** PostgreSQL with [Drizzle ORM](https://orm.drizzle.team/)
- **Caching:** Redis (`ioredis`)
- **Validation:** [Zod](https://zod.dev/)
- **Authentication:** JWT (JSON Web Tokens)
- **Storage:** AWS S3 (or compatible, like MinIO)
- **Logging:** Winston
- **Task Scheduling:** `node-cron`
- **Documentation:** Swagger (Swagger UI)

### Architecture
The project follows a modular architecture:
- `src/modules/`: Contains business modules (e.g., `auth`, `profile`, `salary`, `recruitment`). Each module typically includes:
  - `*.routes.ts`: Express routes.
  - `*.controller.ts`: Request handling and response.
  - `*.service.ts`: Business logic.
  - `*.repo.ts`: Database access.
  - `*.schema.ts`: Zod validation schemas.
- `src/core/`: Shared core functionalities:
  - `auth/`: JWT and session management.
  - `permissions/`: ABAC (Attribute-Based Access Control) and permission services.
  - `workflow/`: Custom workflow engine for approvals and processes.
  - `cache/`: Redis-based caching logic.
  - `middlewares/`: Global and reusable Express middlewares (auth, error handling, rate limiting, audit logging).
- `src/db/`: Database-related files:
  - `schema/`: Drizzle schema definitions.
  - `migrations/`: SQL migration files generated by Drizzle Kit.
  - `seeds/`: Initial data scripts.
- `src/services/`: Shared utility services like `email.service.ts`, `storage.service.ts`, `audit.service.ts`, and `notification.service.ts`.
- `src/jobs/`: Background tasks (e.g., expiry checks, notification workers).

## Building and Running

### Prerequisites
- Node.js (v20+ recommended)
- `pnpm` (Package Manager)
- PostgreSQL
- Redis
- S3-compatible storage

### Environment Variables
Copy `.env.example` to `.env` and fill in the required values:
- `DATABASE_URL`: PostgreSQL connection string.
- `REDIS_URL`: Redis connection string.
- `JWT_SECRET`: Secret for signing JWTs.
- `S3_*`: Configuration for object storage.
- `SMTP_*`: Configuration for email services.

### Development
```bash
# Install dependencies
pnpm install

# Run in development mode (with hot-reloading)
pnpm run dev

# Generate Drizzle migrations
pnpm run db:generate

# Push changes to database
pnpm run db:push

# Open Drizzle Studio (DB GUI)
pnpm run db:studio
```

### Production
```bash
# Build the project
pnpm run build

# Start the built server
pnpm run start
```

## Development Conventions

### Coding Style
- Use TypeScript for all source files.
- Follow the modular structure in `src/modules/`.
- Use Zod for request validation and type inference.
- Use `logger.info`, `logger.error`, etc., for logging instead of `console.log`.
- Handle errors using custom error classes (e.g., `UnauthorizedError`, `BadRequestError`) which are caught by the global `errorHandler`.

### Database
- Define schemas in `src/db/schema/`.
- Use Drizzle ORM for queries.
- Prefer repository pattern (`*.repo.ts`) for complex database operations.

### API Standards
- APIs are prefixed with `/api/v1`.
- Follow RESTful principles.
- Use standard HTTP status codes.
- Use `requestId` middleware for tracing requests.
- Document APIs using JSDoc comments for Swagger.
